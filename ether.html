Ether

Genre: Sound Game

A short production time.

The very short production time (2 months) forced to be efficient with the code, because I wouldnâ€™t have time to do big rewrite it if change in the game design had to be made.

As such at the very start of the project, I outlined the great component of our game:

  • Entity with lots of variable logic to handle
  • A freeform construction system that needs to be 100% reliable.
  • Easy to iterate procedural generation.

So, I chose an architecture inspired by modular pipeline: very atomically made snippets of logic that calls themselves in chains with as few in and out possible.

Build System not mechanics.

The first thing done was creating the play space.

But building it by hand every time we wanted to try a size would have been a loss of valuable time.

This script is responsible for procedural instantiation of a grid and all the game components that require the context of the grid

Sculpt System

Second was the only system that needed to rewrite from its original form, note that the vast majority of methods were already written.

This system is responsible is a group of different pipelines that makes and destroy walls.

To make it scalable in the long-term wall creator was from the get-go written as a library of modular method for each little operation that our walls could carry out.

When Sculptor script was rewritten, the only thing that truly changed was the logic of when and how to call the generic WallCreator methods.

This flexibility created by forward planning in code allowed us to rework one of our core mechanics in less than an afternoon freeing up resources to improve other aspects.
