Magic'Stick

Genre: Beat 'em up

Gameplay Programmer

Using the rather limited visual coding of Construct 3, I created and fully customisable wave manger and AI for the enemy.

The Wave Manager uses a 3-dimensional array stored in a. Son to encode all information needed to create a wave (the position of each spawner, what to spawn and how much). This. Jason is loaded from browser memory on lunch and then read every start of waves to position everything and then call the function that creates an enemy.

The Enemy AI used A*algorithm, where each enemy move as a self-avoiding groups. To achieve this, I need to either:

  • Recalculate the A* map every frame which would kill performance
  • Overwrite the behaviour which would make enemies behave unpredicted.

So, clearly I needed a better solution. Ultimately, I landed on a mix of both:

  • Every 0.5 sec we recalculate the map around the enemy far from view, and every frame we do it for the enemy close to the player. Since the player focus on close enemies, we sacrifice a bit of performance to make them responsive.
  • If an enemy overlap another enemy we overwrite the pathfinding with a biased random walk. While overlapping the enemy do not recalculate the A* map to save on performance.

Level Design:

After coding it, I also designed every wave manually.
